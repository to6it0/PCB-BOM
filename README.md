# PCB-BOM

A Claude Code skill for working with Bills of Materials (BOMs) for PCB projects — create, parse, validate, merge, and cross-reference BOMs exported from Altium, KiCad, or Excel.

## What it does

Invoke with `/bom` in Claude Code. The main workflow is:

| Command | Description |
|---|---|
| `/bom source` | Full sourcing workflow: validates MPNs, resolves connector mating/terminal part numbers + obsolescence status from the manufacturer's own site, computes terminal quantities needed, exports a real `.xlsx` (BOM + Supplier Comparison sheets) plus an interactive `.html` dashboard, and compares EUR price/stock across DigiKey and EU distributors (Mouser, Farnell, RS Components, TME, defaulting to Luxembourg-adjacent locales). Multi-supplier pricing defaults to connectors/mating/terminal parts only — use `/bom source full` to sweep every line item |

See [`SKILL.md`](./SKILL.md) for the full set of commands (`validate`, `jlcpcb`, `mouser`, `digikey`, `merge`, `obsolete`) and their detailed behavior.

## Interactive sourcing dashboard

`/bom source` generates a self-contained `<board name>_Dashboard.html` alongside the `.xlsx` — no server, no build step, just open it in a browser (it's also published as a Claude Artifact for a live preview link). It includes:

- **Sourced BOM table** — sortable and filterable, with status chips distinguishing a resolved mating part number from a connector that mates with a custom cable (no fixed catalog PN)
- **Supplier Comparison table** — grouped by connector (base component → mating housing → terminal), with clickable links per supplier and a distinction between verified data (DigiKey) and search-only links (sites that block automated price/stock extraction)
- **Editable Price, Stock, and Best Price cells** — update a price you found elsewhere, note the actual best-price supplier and its minimum order quantity; edits are highlighted and persist across reloads (stored per-board in the browser), with a one-click reset back to the sourced values
- **Light/dark theme**, self-hosted fonts (works fully offline)

Generated by [`scripts/generate_dashboard.py`](./scripts/generate_dashboard.py) — a reusable function (`generate_dashboard(board_name, sourced_bom, supplier_rows, output_path, ...)`), not a one-off template. See the module's docstring for the exact data schema. Fonts are cached as base64 in [`assets/fonts/`](./assets/fonts/) so it never needs to fetch them at generation time.

## Supported input formats

- Altium BOM exports (`.xlsx` / `.csv`) — typical columns: `Comment`, `Description`, `Designator`, `Footprint`, `LibRef`, `Quantity`, `Manufacturer`, `Manufacturer Part Number`
- KiCad BOM exports
- Generic Excel/CSV component lists

## Usage: from zero to a dashboard

A complete walkthrough for someone who has never used Claude Code before.

### 1. Prerequisites

- macOS 13+, Windows 10 1809+ (or WSL), Ubuntu 20.04+/Debian 10+/Alpine 3.19+
- 4 GB+ RAM
- A Claude.ai account (Free/Pro/Max) or a Claude Console (API) account

### 2. Install Claude Code

Native installer (recommended — updates itself automatically):

| OS | Command |
|---|---|
| macOS / Linux / WSL | `curl -fsSL https://claude.ai/install.sh \| bash` |
| Windows (PowerShell) | `irm https://claude.ai/install.ps1 \| iex` |
| Windows (cmd.exe) | `curl -fsSL https://claude.ai/install.cmd -o install.cmd && install.cmd && del install.cmd` |

Alternatives: `brew install --cask claude-code` (macOS/Linux), `winget install Anthropic.ClaudeCode` (Windows), or `npm install -g @anthropic-ai/claude-code` (requires Node.js 22+).

### 3. First run / login

```bash
claude
```

This opens a browser to log in with your Claude.ai (or Console) account. If the browser can't redirect back automatically (common over SSH or in a container), it shows a code — paste it into the terminal prompt. You'll see `Login successful`; press Enter to land in the interactive session.

### 4. Install this skill

Personal skills live in `~/.claude/skills/<name>/` (macOS/Linux) or `%USERPROFILE%\.claude\skills\<name>\` (Windows) and are picked up in every project. The folder name becomes the slash command, so this repo needs to land in a folder named `bom`:

```bash
# macOS / Linux / WSL
git clone https://github.com/to6it0/PCB-BOM.git ~/.claude/skills/bom

# Windows PowerShell
git clone https://github.com/to6it0/PCB-BOM.git "$env:USERPROFILE\.claude\skills\bom"
```

Prefer it scoped to one project instead? Clone into `.claude/skills/bom` at the project root — same effect, just local to that repo.

No restart needed — Claude Code picks up skills live within a session.

### 5. Run it on a real BOM

1. `cd` into the folder with your board's BOM export (Altium/KiCad `.xlsx` or `.csv`) and run `claude` to start a session there.
2. `/bom` — loads the most recent BOM file in the directory and summarizes it (line items, missing MPNs/manufacturers, duplicates).
3. `/bom source` — the full workflow: validates the BOM, resolves connector mating housings/terminals (or mating cable specs for FFC/FPC connectors) and their lifecycle status straight from the manufacturer's site, then compares price/stock in EUR across DigiKey and EU distributors (Mouser, Farnell, RS, TME). Use `/bom source full` to extend pricing to every line item instead of just connectors.

### 6. What you get

In the same folder:
- `<board name>_Sourcing.xlsx` — a `BOM` sheet and a `Supplier Comparison` sheet
- `<board name>_Dashboard.html` — the interactive dashboard described below; open it directly in a browser, no server needed. Claude also publishes it as a Claude Artifact in the chat for a shareable live-preview link.

See [`SKILL.md`](./SKILL.md) for the full skill definition, including `validate`, `jlcpcb`, `mouser`, `digikey`, `merge`, and `obsolete`.
